# Пакет задач после брейншторма 18-22 мая

**Дата сборки:** 2026-05-22
**Обновлено:** 2026-05-24 (даты выставки уточнены)
**Источник решений:** [`2026-05-18-ui-i-metriki.md`](2026-05-18-ui-i-metriki.md) + чат Neiry 20-22.05 + мысли PM 24.05
**Контекст:** Startup Village **28-29.05.2026**. Стыковка / прогон питча — **27.05 (среда)** в офисе.
**Статус:** Блок A (UI и сценарии) закрыт. Блоки B-C-D-E отложены до post-Village.

## Что изменилось 24.05

- **Дата выставки:** 25.05 → **28-29.05** (стыковка 27.05). Все «горящие» задачи сдвигаются на +3 дня.
- **iOS — НЕ делаем для M2** (решение Кирилла 20.05). Только Android. 1 браслет = 1 телефон.
- **Сценарий стенда (PM 24.05):** 3 браслета (2 анкера + 1 гость), калибровка 10-15 сек, drill-down с опциональной кнопкой «Тест Баевский».
- **Stress Index:** Fast (NCI на плазме) + Slow (Баевский ИН в TG-отчёте, фильтр R-R 5%).
- **Корпоративный дашборд:** 3 таба — Спорт / Офис-команда / Драйверы-водители (M3+).
- **Линейка устройств:** Base (на руках, только тесты) / Health VANTA (июнь, ECG) / Sport VIGOR (июнь, GPS).
- **V2 features:** анализатор калорий по фото, female cycle, служба заботы для пенсионеров — см. `roadmap/v2-features.md`.

Брейншторм закрыл 5 подблоков по UI и сценариям M2:
1. **Wow-сценарий стенда** — Струп + перформер-офисник на беговой дорожке, параллельные браслеты у гостя

4. **Mobile UI** — берём M1 wireframes как baseline, чистим дев-жаргон
5. **Neiry Unite (HR-дашборд)** — Никитин v4 как baseline + 3 локальные правки

### Кирилл (backend + mobile)

**Горящее (до 28.05):**
- [ ] Собрать APK с UI по `docs_web/wireframes/m1/*.html` (не дев-сборку) для 3 gateway-телефонов
- [ ] Endpoint `POST /admin/seed-rr {user_id, rr_intervals[], date}` (один эндпоинт, ~30 мин работы)
- [ ] Расчёт RMSSD на backend по Task Force HRV Standards 1996 (формула в [`metrics/hrv.md`](../metrics/hrv.md))
- [ ] **NCI Fast** (для плазмы): `100 − (BPM − Baseline) × 2`, clamp [0..100], цвета 🟢80-100 / 🟡50-79 / 🔴0-49
- [ ] **Баевский ИН Slow** (для TG-отчёта): `AMo / (2·Mo·MxDMn)`, окно 3-5 мин, фильтр R-R-артефактов 5%
- [ ] 5-й модуль на карточке плазмы — HRV-блок (2 состояния: есть данные / нет данных)
- [ ] **Архитектура 1 браслет = 1 телефон закрепляется** (решение Кирилла 20.05). Подтвердить: 3 BLE-connection с одного Android — НЕ держим, нужно 3 разных телефона.
- [ ] Подтверждено: **3 одновременных гостя** на плазме (2 анкера + 1 гость).
- [ ] Drill-down с карточки + опциональная кнопка «Тест Баевский» (3 мин неподвижно + Stroop + дорожка ≈ 5 мин)

**M3 (после выставки):**
- [ ] 4 типа event-алертов в Celery-task:
  - **Sleep Deficit** (Sleep <6ч 3+ дня подряд)
  - **Early Illness Alert** (RHR↑5 + HRV↓10 за 2 дня — формула в [`metrics/rhr.md`](../metrics/rhr.md))
  - **Burnout Trend Warning** (Burnout Risk вырос ≥15 п.п. за 14d + пересёк порог 60%)
  - **Recovery Comeback** (переход Readiness из красного → зелёного)
- [ ] Backend для Neiry Unite по Никитину дизайну v4 (taxonomy метрик в [Vitro Registry](../metrics/vitro-metrics-registry.md))
