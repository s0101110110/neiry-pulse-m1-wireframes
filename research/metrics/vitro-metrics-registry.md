# Vitro Metrics Registry v0.2

_Источник: Neiry_Vitro_Metrics_Registry_v0.2.xlsx_
_Обработано: 2026-05-17_

Полный реестр метрик умных браслетов Neiry на платформе HBandSDK (Veepoo).
Покрывает линейку: **VITRO** (Base) · **VANTA** (Health) · **VIGOR** (Sport).

---

## Уровни метрик (L0–L4)

| Уровень | Название | Где считается |
|---------|----------|---------------|
| L0 | Сырые данные с сенсора | Железо браслета |
| L1 | Первичные прямые метрики | Железо / SDK |
| L2 | Производные (научные формулы) | SDK / Backend |
| L3 | Композитные индексы (проприетарные) | Backend / SDK |
| L4 | Предиктивная аналитика (ML) | Backend ML |

## Маркеры доверия

| Маркер | Уровень | Что означает |
|--------|---------|--------------|
| 🟢 G | Достоверно | Клинически валидировано, можно показывать без оговорок |
| 🟡 Y | Условно | Метод корректный, но зависит от условий — показывать с пояснением (i) |
| 🟠 O | Сомнительно | Косвенное измерение — только за переключателем «Дополнительно» с дисклеймером |
| 🔴 R | Маркетинг | Физически невозможно — удалить из дефолтных пресетов |

# ГРУППА: Cardio — Сердечно-сосудистая система

## 1. Частота сердечных сокращений (ЧСС / HR)

**Что измеряет:** Мгновенное значение пульса — количество ударов сердца в минуту.
**Единицы:** уд/мин (bpm); диапазон 30–200
**Норма:** 60–100 в покое у взрослых
**Сенсор:** Зелёный PPG (фотоплетизмография 525 нм)
**Уровень:** L1 — Primary
**Маркер доверия:** 🟢 G — клинически валидировано, ±2–5 bpm в покое
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Фундамент всех кардио-расчётов. Используется для HRV, Stress Score, Recovery, Calories, HR-зон, пульса покоя.
**Считается:** На MCU браслета (firmware Veepoo) в реальном времени, 1 Hz.
**Открытые вопросы:** Есть ли signal quality indicator?

---

## 2. R-R интервалы (RR / IBI)

**Что измеряет:** Временные промежутки между последовательными ударами сердца. Фундамент всех HRV-метрик.
**Единицы:** мс; диапазон 300–2000
**Норма:** зависит от ЧСС; для 60 bpm ≈ 1000 мс
**Сенсор:** Зелёный PPG → peak detection
**Уровень:** L1 — Primary
**Маркер доверия:** 🟡 Y — точность ±20–40 мс (PPG-25Hz) vs ±1–2 мс (ЭКГ). Для wellness HRV приемлемо.
**Доступность:** VITRO ✓ (TBD) · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Входное значение для всех HRV-метрик (RMSSD, SDNN, pNN50, SD1/SD2, LF/HF).
**Считается:** На MCU браслета.
**Открытые вопросы:** 🔴 КРИТИЧНО (Q1): получаем ли мы массив RR из SDK?

---

## 3. Пульс покоя (RHR)

**Что измеряет:** Минимальный пульс за период покоя (ночное окно или раннее утро). Базовый индикатор кардиоздоровья.
**Единицы:** уд/мин; диапазон 40–80
**Норма:** 50–70 у здоровых взрослых; снижение — признак хорошей физической формы
**Сенсор:** HR (L1) → ночные семплы
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — научно валидированный маркер. Тренд критичнее абсолюта.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Входит в Recovery Score, Cardiovascular Fitness, долгосрочные тренды здоровья.
**Считается:** Backend, раз в сутки (утренний пересчёт по hr_samples).

---

## 4. Максимальный пульс (Max HR)

**Что измеряет:** Максимальное значение пульса за период (тренировка / день).
**Единицы:** уд/мин; теоретический максимум = 220 − возраст
**Норма:** индивидуально; зависит от возраста и уровня тренированности
**Сенсор:** HR (L1) → семплы за окно
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — простая агрегация; точность зависит от качества HR-измерения.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** HR Zones (тренировочные зоны), расчёт VO₂max.
**Считается:** Backend, раз в тренировку или раз в сутки.

---

## 5. Пульсовые зоны (HR Zones)

**Что измеряет:** Распределение времени по 5 зонам интенсивности (Z1–Z5) на основе % от HRmax.
**Единицы:** минут в каждой зоне; Z1: 50–60%, Z2: 60–70%, Z3: 70–80%, Z4: 80–90%, Z5: 90–100% HRmax
**Норма:** зависит от цели тренировки
**Сенсор:** HR (L1) + HRmax
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — международный стандарт ACSM/IOC.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓ (с GPS точнее)
**Как используется в Neiry Pulse:** Training Load, TRIMP, отображение тренировочных зон в реальном времени.
**Считается:** SDK или Backend, реальное время во время тренировки.

---

## 6. Восстановление пульса (HRR)

**Что измеряет:** Скорость снижения пульса после нагрузки. HRR за 1 минуту — кардиомаркер.
**Единицы:** уд/мин (снижение за мин); норма: >12 уд/мин за 1 мин — хорошо, <12 — слабо
**Норма:** >12 уд/мин снижения за первую минуту восстановления
**Сенсор:** HR (L1) после окончания тренировки
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — научно валидированный маркер кардиофитнеса (Cole et al, NEJM).
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Cardiovascular Fitness Index, Recovery Score.
**Считается:** Backend, по окончании каждой тренировки (batch).

# ГРУППА: HRV — Вариабельность сердечного ритма

## 7. HRV — RMSSD

**Что измеряет:** Root Mean Square of Successive Differences — квадратный корень из среднего квадратов разностей соседних RR-интервалов. Отражает парасимпатическую (вагусную) активность нервной системы.
**Единицы:** мс; типично 15–80 у здоровых взрослых
**Норма:** снижение от индивидуальной базовой линии = стресс/усталость; рост = хорошее восстановление
**Сенсор:** RR-интервалы (L1)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — метод корректный, но точность ограничена 25 Hz PPG (завышенная RMSSD). Тренд достоверен.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Ключевой входной параметр для Stress Score, Recovery Score, Burnout Risk Index. Также используется для Closed-Loop Recovery с tVNS.
**Считается:** SDK native (libnative-lib.so) или Backend, батчами каждые N минут или ночью.
**Открытые вопросы:** Q2: что именно возвращает `bod_hrv_char_t` в SDK?

---

## 8. HRV — SDNN

**Что измеряет:** Standard Deviation of NN-intervals — стандартное отклонение всех RR-интервалов. Отражает общую вариабельность (парасимпатика + симпатика).
**Единицы:** мс; типично 50–200 для 24-часового окна
**Норма:** >100 мс — хорошо; <50 мс — повышенный риск кардиопроблем
**Сенсор:** RR-интервалы (L1)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — менее чувствителен к ошибкам PPG чем RMSSD, но всё равно wellness-уровень.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Часть Stress Score, долгосрочные тренды Recovery.
**Считается:** SDK native или Backend.

---

## 9. HRV — pNN50

**Что измеряет:** Процент RR-интервалов, разница которых с предыдущим превышает 50 мс. Отражает парасимпатическую активность.
**Единицы:** %; типично 5–40 у здоровых взрослых
**Норма:** выше = лучше парасимпатический тонус
**Сенсор:** RR-интервалы (L1)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — при низком разрешении PPG пороговая метрика особенно чувствительна к шуму.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Дополнительный компонент для Stress Score (опционально).
**Считается:** Backend (если есть RR) или SDK.

---

## 10. SD1 (Пуанкаре)

**Что измеряет:** Стандартное отклонение перпендикулярно линии идентичности на эллипсе Пуанкаре. Краткосрочная вариабельность.
**Единицы:** мс; коррелирует с RMSSD (≈ RMSSD/√2)
**Норма:** аналогично RMSSD
**Сенсор:** RR-интервалы (L1), метод построения графика Лоренца
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — стандарт визуализации HRV.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Veepoo использует именно эту метрику для Stress Score и HRV Visualization.
**Считается:** SDK native (HRV_LorenzDetection в libnative-lib.so).

---

## 11. SD2 (Пуанкаре)

**Что измеряет:** Стандартное отклонение вдоль линии идентичности на эллипсе Пуанкаре. Долгосрочная вариабельность.
**Единицы:** мс; обычно SD2 > SD1
**Норма:** соотношение SD1/SD2 отражает симпатовагальный баланс
**Сенсор:** RR-интервалы (L1)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — менее чувствителен к низкому разрешению чем SD1.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Индикатор вегетативного баланса через соотношение SD1/SD2.
**Считается:** SDK native.

---

## 12. LF/HF — частотный анализ ВСР

**Что измеряет:** Соотношение мощностей Low Frequency (0.04–0.15 Hz) и High Frequency (0.15–0.4 Hz) диапазонов RR-интервалов. Индикатор симпатовагального баланса.
**Единицы:** безразмерный; типично 0.5–2.0
**Норма:** значения сильно зависят от метода измерения и условий
**Сенсор:** RR-интервалы (L1) → FFT-анализ
**Уровень:** L2 — Derived
**Маркер доверия:** 🟠 O — при 25 Hz PPG частотный анализ ненадёжен. Требует >256 Hz разрешения RR.
**Доступность:** VITRO ✓ (только если есть RR) · VANTA ✓ (TBD) · VIGOR ✓ (TBD)
**Как используется в Neiry Pulse:** Опциональный компонент Stress Score / Recovery Score.
**Считается:** Backend (scipy.signal.welch на Python).
**Открытые вопросы:** Q1 + стоит ли считать вообще при PPG-25Hz?

# ГРУППА: Blood — Параметры крови и ЭКГ

## 13. Сатурация кислорода (SpO₂)

**Что измеряет:** Процент гемоглобина, насыщенного кислородом.
**Единицы:** %; диапазон 70–100
**Норма:** 95–100% — норма; 90–94% — сниженная; <90% — критично
**Сенсор:** Красный (660 нм) + ИК (940 нм) PPG
**Уровень:** L1 — Primary
**Маркер доверия:** 🟡 Y — точность ±2–4% vs медицинский пульсоксиметр. Для трендов, не для медицинских решений.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Sleep Quality Score (детекция апноэ), мониторинг при горных условиях (VIGOR).
**Считается:** На MCU браслета, on-demand или каждые N минут.
**Открытые вопросы:** Q5: есть ли confidence score? Коррекция на тон кожи?

---

## 14. Артериальное давление — публичный режим (BP-public)

**Что измеряет:** Систолическое / диастолическое давление без калибровки. Чистый алгоритм Veepoo на основе PPG.
**Единицы:** мм рт. ст.; систолическое 60–300, диастолическое 20–200
**Норма:** систолическое <120, диастолическое <80 — оптимально
**Сенсор:** PPG (волновая форма) + ML
**Уровень:** L1 — Primary
**Маркер доверия:** 🟠 O — без калибровки манжетой PPG-BP это wellness-маркетинг, не измерение.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Не используется в производных метриках из-за низкого доверия.
**Считается:** На MCU, on-demand (50–60 сек).
**Открытые вопросы:** Удалить из default presets или показывать с явным дисклеймером?

---

## 15. Артериальное давление — приватный режим (BP-private)

**Что измеряет:** Систолическое / диастолическое давление с индивидуальной калибровкой по эталонной манжете.
**Единицы:** мм рт. ст.; точность ±10–15 при свежей калибровке
**Норма:** систолическое <120, диастолическое <80 — оптимально
**Сенсор:** PPG + калибровочная константа пользователя
**Уровень:** L1 — Primary
**Маркер доверия:** 🟡 Y — приемлемо для тренда при калибровке раз в месяц. Не заменяет тонометр.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Cardiovascular Risk Index для Care-сегмента.
**Считается:** На MCU. Требует предварительную калибровку.
**Открытые вопросы:** Q9: процедура калибровки, как часто обновлять?

---

## 16. Одноотведенная ЭКГ (ECG)

**Что измеряет:** Электрическая активность сердца, отведение Lead I. 30 секунд по запросу.
**Единицы:** мВ; дискретизация ~512 Hz
**Норма:** визуальный анализ ритма; клиническая оценка — компетенция врача
**Сенсор:** ЭКГ-электроды (1 на корпусе + 1 на пальце)
**Уровень:** L1 — Primary
**Маркер доверия:** 🟢 G — 512 Hz Lead I — клинически приемлемое разрешение для детекции аритмий. Apple Watch FDA-cleared на аналогичной архитектуре.
**Доступность:** VITRO нет · VANTA ✓ · VIGOR нет
**Как используется в Neiry Pulse:** ECG-32 диагностика, точные RR-интервалы (для VANTA premium), детекция AFib.
**Считается:** Raw на MCU, DSP-обработка в SDK (libnative-lib.so), 30-секундная сессия.
**Открытые вопросы:** Q3: сертификация, точность AFib detection.

---

## 17. RR-интервалы из ЭКГ (ECG-RR)

**Что измеряет:** RR-интервалы с медицинской точностью (±1–2 мс) из ЭКГ-записи. Золотой стандарт для HRV.
**Единицы:** мс; точность ±1–2 мс (vs ±20–40 мс у PPG)
**Норма:** аналогично RR от PPG, но значительно точнее
**Сенсор:** ECG ADC raw (L0) → R-peak detection
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — золотой стандарт RR-измерения.
**Доступность:** VITRO нет · VANTA ✓ · VIGOR нет
**Как используется в Neiry Pulse:** Высокоточный HRV для медицинских пресетов VANTA.
**Считается:** SDK native или Backend, только во время 30-секундной ECG-записи.

---

## 18. Время пульсовой волны (PTT)

**Что измеряет:** Время от R-пика ЭКГ до пика PPG-волны на запястье. Косвенная оценка давления через упругость сосудов.
**Единицы:** мс; типично 200–400
**Норма:** уменьшение PTT коррелирует с ростом систолического давления
**Сенсор:** ECG + PPG синхронно
**Уровень:** L2 — Derived
**Маркер доверия:** 🟠 O — без индивидуальной калибровки даёт только грубую оценку.
**Доступность:** VITRO нет · VANTA ✓ (с ECG) · VIGOR нет
**Как используется в Neiry Pulse:** Альтернативный источник для BP-private mode.
**Считается:** SDK native (PTT API, раздел 4.4), во время одновременной ECG+PPG сессии.

# ГРУППА: Activity — Дневная активность

## 19. Количество шагов (Steps)

**Что измеряет:** Количество шагов за период (день / час).
**Единицы:** шт.; 0–50000+ за день
**Норма:** цель WHO — 7000–10000 шагов/день
**Сенсор:** IMU акселерометр (3 оси)
**Уровень:** L1 — Primary
**Маркер доверия:** 🟢 G — точность ±5–10% vs медицинский педометр.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Calories, Active Minutes, Sedentary Alert, Daily Activity Score, Distance.
**Считается:** Непрерывно на MCU, агрегация раз в минуту; синхронизация батчами.

---

## 20. Пройденная дистанция (Distance)

**Что измеряет:** Расстояние, пройденное за период.
**Единицы:** м; 0–50000+ за день
**Норма:** зависит от цели
**Сенсор:** Steps + длина шага (VITRO/VANTA) или GPS (VIGOR)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G (с GPS) / 🟡 Y (без GPS — ±10–20%)
**Доступность:** VITRO ✓ (без GPS) · VANTA ✓ (без GPS) · VIGOR ✓ (с GPS)
**Как используется в Neiry Pulse:** Calories, Pace, отображение дистанции тренировки.
**Считается:** Hardware без GPS; Backend с GPS.

---

## 21. Активные минуты (MVPA)

**Что измеряет:** Время активности средней и высокой интенсивности за день (WHO: 150 мин/нед = 21+ мин/день).
**Единицы:** минут; цель WHO ≥21 мин/день
**Норма:** ≥150 минут умеренной или ≥75 минут интенсивной активности в неделю (ВОЗ)
**Сенсор:** HR + Steps + IMU
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — стандарт WHO/ACSM (формула Карвонена).
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Daily Activity Score, Sedentary Alert (обратная связь), WHO compliance.
**Считается:** Backend, раз в сутки.

---

## 22. Сожжённые калории (Calories / TDEE)

**Что измеряет:** Общий расход энергии за день: базовый обмен (BMR) + активность + термический эффект пищи.
**Единицы:** ккал; зависит от пола, возраста, веса, уровня активности
**Норма:** индивидуально; типично 1800–2500 ккал/день для взрослых
**Сенсор:** Steps + HR + профиль пользователя
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — PPG-браслеты дают ±15–25% погрешности по калориям.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Energy balance, отображение в Daily Dashboard.
**Считается:** Hardware или Backend (Mifflin-St Jeor BMR + METs × время × вес).

---

## 23. Метаболический эквивалент (MET)

**Что измеряет:** Интенсивность активности относительно покоя. 1 MET = 3.5 мл O₂/кг/мин (базовый обмен).
**Единицы:** MET; 1 (покой) — 20 (max); ходьба ≈ 3–4 MET, бег ≈ 7–12 MET
**Норма:** зависит от вида активности
**Сенсор:** HR + тип активности
**Уровень:** L2 — Derived

## 24. Каденс (Cadence)

## 25. Темп (Pace)

# ГРУППА: Sleep — Сон

## 26. Общая длительность сна (TST)

**Что измеряет:** Общее время сна за ночь (от засыпания до пробуждения, минус периоды бодрствования).

## 27. Фазы сна (Sleep Stages)

## 28. Время засыпания (SOL)

**Что измеряет:** Время от момента, когда пользователь лёг, до фактического засыпания.
**Единицы:** минут; норма 10–20

**Считается:** Backend, раз за ночь.

## 29. Бодрствование после засыпания (WASO)

**Что измеряет:** Суммарное время пробуждений во время ночи (после первого засыпания).
**Единицы:** минут; норма <30
**Норма:** <30 минут; увеличение — признак нарушения качества сна
**Сенсор:** Sleep events (периоды пробуждений)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟡 Y — запястный датчик пропускает тихие пробуждения без движения.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Sleep Efficiency, Sleep Score.
**Считается:** Backend, раз за ночь.

## 30. Эффективность сна (SE)

**Что измеряет:** Процент от времени в кровати, проведённого во сне. Гигиена сна.
**Единицы:** %; норма >85
**Норма:** ≥85% — хорошо; <85% — нарушение
**Сенсор:** TST + Time in Bed (через SOL + WASO)
**Уровень:** L2 — Derived
**Маркер доверия:** 🟢 G — стандартная формула сомнологии.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Sleep Score, Recovery Score, компонент Neiry Daily Score.
**Считается:** Backend: SE = TST / (TST + SOL + WASO) × 100.

## 36. TRIMP (Training Impulse)

## 37. Тренировочная нагрузка: ATL / CTL / ACR

**Что измеряет:** ATL — скользящее среднее TRIMP за 7 дней (острая нагрузка); CTL — за 28 дней (хроническая); ACR — их соотношение.

**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓

**Считается:** Backend, раз в сутки.

**Что измеряет:** Риск профессионального выгорания. Уникальная метрика для Corporate-сегмента.
**Единицы:** 0–100; >70 — высокий риск
**Норма:** <40 — низкий риск
**Сенсор:** HRV + Sleep + Stress (тренды 14 дней)
**Уровень:** L3 — Composite (Neiry)
**Маркер доверия:** 🟡 Y — концепция проверяется на корпоративных пилотах. Требует анонимизации для HR-аналитики.
**Доступность:** VITRO ✓ · VANTA ✓ · VIGOR ✓
**Как используется в Neiry Pulse:** Corporate HR analytics (агрегированно по отделам, без привязки к ФИО).
**Считается:** Backend, раз в сутки. Формула: `Burnout RI = weighted_sum(HRV_decline_14d, sleep_deficit_14d, daily_stress_mean_14d)`.

**Как используется в Neiry Pulse:** Athlete coaching, корректировка тренировочного плана.
**Считается:** Backend утром. Формула: `Readiness = 0.5×Recovery + 0.3×HRV_today + 0.2×sleep_last_night`.
