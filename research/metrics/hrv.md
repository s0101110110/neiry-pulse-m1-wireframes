# HRV / ВСР — Heart Rate Variability / Вариабельность Сердечного Ритма

_Последнее обновление: 2026-05-22_

## Что это простыми словами

**ВСР — это микро-разница во времени между соседними ударами сердца.**

Сердце **не** стучит как метроном. Между ударами:
- сейчас 820 мс
- через удар — 790 мс
- через ещё один — 830 мс

Эта микро-разница и есть ВСР.

### Парадокс ВСР
- **Ровный пульс** (интервалы одинаковые) = **плохо**. Тело в боевом режиме.
- **«Прыгающие» интервалы** = **хорошо**. Тело восстанавливается.

Почему так:
- Парасимпатика (блуждающий нерв, «отдых») → интервалы разные → высокая ВСР
- Симпатика («стресс») → интервалы выравниваются → низкая ВСР

## Зачем меряем
- **Recovery / Readiness** — все wellness-носимые (Apple Watch, Polar, WHOOP, Oura, Garmin) строят утренний «Recovery Score» на ВСР
- **Burnout / выгорание** — затяжное падение ВСР = ранний признак
- **Early Illness Alert** — ВСР падает за 2-3 дня до симптомов ОРВИ
- **Стресс-тренды** на длинных горизонтах

## RMSSD — главная метрика ВСР

Расшифровывается «Root Mean Square of Successive Differences».
По-русски: «корень из среднего квадрата разности соседних RR-интервалов».

**Формула:**
```
1. Берём массив RR-интервалов: [820, 850, 810, 830, 845, ...] мс
2. Считаем разности соседних: [+30, −40, +20, +15, ...]
3. Квадраты разностей: [900, 1600, 400, 225, ...]
4. Среднее по квадратам: например, 781
5. Корень: sqrt(781) ≈ 28 мс
```

RMSSD = 28 мс.

**Шкала для здорового взрослого:**
| RMSSD | Состояние |
|-------|-----------|
| **60–100 мс** | Отлично, спортивная форма / хорошее восстановление |
| **30–60 мс** | Норма |
| **15–25 мс** | Звоночек: выгорание / переутомление / начинающаяся болезнь |
| **< 15 мс** | Тревожно, к врачу |

Норма падает с возрастом (~50 мс в 25 лет → ~30 мс в 60 лет), у мужчин обычно ниже, чем у женщин одного возраста.

## Другие HRV-метрики (контекст)
- **SDNN** — общая ВСР (включает и парасимпатику, и симпатику). Хорош для долгосрочных трендов
- **pNN50** — процент RR-пар, отличающихся > 50 мс. Альтернатива RMSSD
- **SD1, SD2** — оси эллипса Пуанкаре (Lorenz plot)
- **LF/HF** — частотный анализ (Low Frequency / High Frequency), требует > 5 мин записи

**Для Neiry Pulse главная метрика — RMSSD**, потому что:
1. Чувствителен к краткосрочным изменениям парасимпатики (то, что нужно для Recovery)
2. Стандарт в wellness-носимых (Polar Nightly Recharge, Apple Heart Study)
3. Простой расчёт из RR-массива

## Где считается в архитектуре Neiry

| Слой | Что делает |
|------|-----------|
| Браслет (Veepoo MCU) | Детектирует R-пики из PPG, формирует RR-массив, хранит в памяти 64 МБ |
| HBandSDK | Метод `readRRIntervalByDay` отдаёт RR-массив за выбранный день (Q1 закрыт 16.05) |
| Mobile app | Через BLE-bridge вызывает `readRRIntervalByDay`, упаковывает в JSON, шлёт на backend |
| Backend (FastAPI + Celery) | Парсит JSON, считает RMSSD/SDNN/pNN50 по Task Force HRV Standards 1996, кладёт в БД |
| Дашборд/плазма/mobile UI | Читает готовый RMSSD из БД, показывает с интерпретацией |

**Решение от 18.05:** HRV считается **на нашем backend** (не на устройстве, не в SDK), чтобы:
- иметь полный контроль над формулой;
- не зависеть от закрытой Veepoo native-библиотеки `libnative-lib.so` (метод `HRV_LorenzDetection`, состав `bod_hrv_char_t` не раскрыт);
- легко обновлять алгоритм без OTA прошивки.

## Ограничения HBandSDK для HRV

- Метод `readRRIntervalByDay` отдаёт RR **исторически батчем** при синхронизации (не tick-by-tick live)
- Значит: на стенде M2 «real-time HRV» невозможен. Только архивный HRV «прошлой ночи»
- Для real-time стресса используется Wow-хак из ΔBPM от baseline (см. `brainstorms/2026-05-18-ui-i-metriki.md`)

## Окно расчёта (рекомендация Никиты для Neiry Unite)

**HRV считается за ночное окно 00:00–07:00.** Причины:
- Минимум артефактов движения и стресса
- Сопоставимые данные день-к-дню (одни и те же часы)
- Все референсы (Polar Nightly Recharge, WHOOP Recovery, Garmin Body Battery) используют похожее окно

Открытый вопрос (брейншторм 18.05): можно ли расширить окно для людей со сменным графиком (диспетчеры, водители, машинисты)? Например, «7 часов сна, найденных по motion + HR». Это потребует honest sleep detection.
