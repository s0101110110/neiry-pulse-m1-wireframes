Мульти-агентная система Neiry

PM-оркестратор + 3 специализированных агента + общий центр знаний

🎼
PM / Оркестратор
docs_web/
+ знает весь контекст
+ делегирует задачи
+ обновляет KB
🎨
UX/UI Дизайнер
UI_assets/
+ UI-киты, ассеты
+ создаёт wireframes
+ HTML/CSS макеты
💻
Frontend Dev
dashboard/
+ стэк фронтенда
+ реализует код
+ по ТЗ от PM
🔬
R&D Researcher
research/
+ железо, SDK
+ простым языком
+ веб-поиск

🧠 Общий центр знаний — knowledge-base/ (Obsidian)

📄 PROJECT_STATE.md
живой снимок проекта
📋 tasks/
задачи по участникам
🏁 milestones/
M1, M2, M3 статус
⚡ decisions/
зафиксированные решения
❓ open-questions/
нерешённые вопросы
📥 INBOX/
зона сброса материалов
↑ Все агенты читают отсюда. Только PM и R&D пишут.

🔄 Типичный workflow

1. 📥 Кладёшь транскрипцию встречи в INBOX/
2. 🎼 PM обрабатывает → обновляет tasks/, decisions/, PROJECT_STATE.md
3. 🎼 PM пишет бриф в dashboard/BRIEF.md для Frontend Dev
4. 💻 Frontend Dev читает бриф → реализует код
5. 🎨 Если нужен UI — PM делегирует субагента в UI_assets/
6. 🔬 Если вопрос по железу — PM запускает R&D Researcher с веб-поиском

📁 Структура папок

NEIRY/
  📚 knowledge-base/  ← Obsidian Vault (общий для всех)
  🎼 docs_web/  ← PM-агент живёт здесь
  🎨 UI_assets/  ← UX/UI дизайнер
  💻 dashboard/  ← Frontend разработчик
  🔬 research/  ← R&D researcher (новая папка)
  📱 mobile/  ← Кирилл (не трогаем)
  ⚙️ backend/  ← Кирилл (не трогаем)
