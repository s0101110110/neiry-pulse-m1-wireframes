Как организовать экраны?

M1: 4 мобильных экрана + 2 экрана дашборда

A

Один HTML-файл — все экраны вместе

Все экраны лежат в одном файле горизонтально или вертикально. Открыл — видишь весь флоу целиком. Удобно для ревью и экспорта в PDF.

Плюсы

  • Один файл — легко шарить
  • Флоу виден целиком

Минусы

  • Нельзя открыть «только логин»
B

Отдельный файл на каждый экран ★

Каждый экран — отдельный HTML. Кирилл открывает нужный, копирует структуру. Просто называется: m1-login.html, m1-ble-pairing.html и т.д.

Плюсы

  • Удобно разработчику
  • Чисто и атомарно

Минусы

  • Флоу нужно смотреть по файлам
C

Index-файл с навигацией между экранами

Отдельные HTML + один index.html с превью всех экранов и кликабельными ссылками. Можно переходить между экранами как в реальном приложении.

Плюсы

  • Интерактивный флоу
  • Идеально для показа Никите

Минусы

  • Чуть больше работы
