---

## 1. Контекст и цель

«Внутренняя страница» карточки гостя/якоря, открывается на админ-ноуте кликом по соответствующей карточке на kiosk-плазме. Полноэкранный 1920×1080, тёмная тема — единый язык со стендом.

**Главная ценность:** в одном экране админ видит и **live-метрики** (что сейчас), и **предиктивные метрики** (что говорит модель), и может запустить **тест Баевского** прямо отсюда, а финалом — забрать **QR с отчётом** для гостя.

**Не путать:** drilldown — это NOT плазма (плазма = `kiosk-v2.html` со стеной из 3 карточек). Drilldown открывается отдельной вкладкой/окном на админ-ноуте.

---

## 2. Технический стек

- HTML5 + Tailwind CSS (CDN) + inline SVG для всех графиков, gauge, иконок
- vanilla JS для: state-switcher (dev-bar), fake live BPM (sin-волна), Bayevsky-таймеры, ECG-анимация, переключение мокаппов 3 персонажей
- Semantic CSS-переменные shadcn (`--background`, `--foreground`, `--card`, `--primary`, `--success`, `--warning`, `--destructive`, `--muted`, `--border`)
- Hex напрямую только для wine accent `#831843` (HSL 333 76% 30%)
- Шрифты: **Space Grotesk** (UI), **Onest** 700 letter-spacing -0.02em (hero-числа), **Geist Mono** (data, метки, session-ID)
- Никаких React/Vue/JSX, никаких inline-стилей `style="..."`, никаких внешних JS-зависимостей кроме Tailwind CDN и Google Fonts

## 5. Левая колонка — LIVE (916×~872)

Сверху вниз. Все блоки на всю ширину 916px (кроме последнего ряда с 2 кнопками).

### 5.1. BPM hero (240px)

```
        ♥  82                BPM
              ● подключен
```

- Иконка сердца ♥ (40px) — выровнена по baseline числа
- Число `82` — Onest 700, **160px**, цвет foreground, моноширинный feel через `font-variant-numeric: tabular-nums`
- Сокращение `BPM` — Geist Mono 24px, muted-foreground, справа от числа (subscript)
- Микро-чип `● подключен` под числом — Geist Mono 14px, цвет success-foreground, точка ● цветом success (зелёный)
- Анимация: число «дышит» (плавный scale 1.0 → 1.02 → 1.0 по pulse каждую секунду)

### 5.2. ECG plashka (140px)

```
┌──────────────────────────────────────────────┐
│ ECG ∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿∿                  │
└──────────────────────────────────────────────┘
```

- Rounded rectangle, `border-border`, `bg-card`, padding 16px
- Inline SVG: ширина 884px, высота 108px, viewBox корректный
- Линия `stroke-primary` (wine) или `stroke-success` (зелёный, по дефолту) — толщина 2px
- Pattern: 8-12 «PQRST»-комплексов, форма реалистичного ECG (P-волна → QRS-комплекс → T-волна)
- Левый край подпись `ECG` (Geist Mono 12px muted)
- Анимация: горизонтальный scroll влево (как осциллограф), 30 кадров в сек, бесшовный loop через два sibling-узора

### 5.3. Stress + IMU круги (320px, ряд 50/50)

```
┌─────────────────┬───────────────┐
│   ◯ 41          │    ◎ IMU      │
│   стресс        │               │
│   ● повышен     │   ● покой     │
└─────────────────┴───────────────┘
```

**Stress (левый круг):**
- Круг 240×240px, простая декоративная рамка `stroke-border` 2px, **без** заливки/gauge внутри
- В центре число `41` — Onest 700, 88px, цвет foreground
- Под числом подпись `стресс` — Space Grotesk 18px, muted-foreground
- Под кругом микро-чип `● повышен` — Geist Mono 14px, цвет = warning-foreground, точка warning (амбра)
- Цвет светофора **передаётся только через текст-чип**, не через заливку круга

**IMU (правый круг):**
- Круг 240×240px, **концентрические декоративные кольца** — 4-5 колец внутри, толщиной 1-2px, цвет `border` с разной opacity (визуальный «радар»)
- В центре подпись `IMU` — Space Grotesk 20px, foreground
- Под кругом микро-чип `● покой` — Geist Mono 14px, цвет = success-foreground

### 5.4. Кнопки СТОП + ТЕСТ (80px, ряд 50/50)

```
┌─────────────────┬───────────────┐
│   [ СТОП ]      │   [► ТЕСТ ]   │
└─────────────────┴───────────────┘
```

- Высота кнопок 64px, gap 16px между ними
- **СТОП:** ghost-стиль (transparent + border-border + foreground), Space Grotesk 18px, padding x-32
- **ТЕСТ:** primary fill (wine `#831843` background + white foreground), Space Grotesk 18px, иконка ► 16px слева
- Hover: СТОП → border-foreground; ТЕСТ → background lighter wine

### 5.5. Микро-чипы — общий стиль

Все 3 чипа (`● подключен / ● повышен / ● покой`) — одинаковый pattern:
- Текст Geist Mono 14px UPPERCASE
- Точка ● — circle 8px, цвет по семантике (success/warning/destructive)
- Контейнер: inline-flex, gap 8px, padding 4×12, rounded-full, `bg-muted/50`
- Цвета:
  - `● подключен` → точка success-foreground (зелёный)
  - `● потеря связи` → точка warning-foreground (амбра) — sub-state при обрыве BLE
  - `● повышен` → точка warning-foreground (амбра)
  - `● высокий` → точка destructive-foreground (красный)
  - `● норма` → точка success-foreground (зелёный)
  - `● покой` → точка success-foreground (зелёный)

### 9.3. Sub-state «потеря связи» (внутри активной сессии)

При обрыве BLE во время `&session=active`:
- **Live-виджеты** (BPM, ECG, Stress, IMU): замораживают **последнее** валидное значение
- BPM микро-чип: `● подключен` → `● потеря связи` (амбра)
- ECG: линия превращается в flat-line по последнему уровню
- **Predictive-виджеты** не трогаем (они на исторических данных, не на live-stream)
- Без баннера сверху (минимально-инвазивно)

## 11. State machine

7 состояний:

```
empty (no session)
  ↓ (URL получает &session=active)
drilldown ←─────────────────────────────────┐
  │  ↓ (клик «ТЕСТ»)                        │
  │  phase-1 (Покой 3 мин)                  │
  │  ↓ (auto после 3 мин или dev-skip)      │
  │  phase-2 (Stroop 1 мин)                 │
  │  ↓ (auto после 1 мин или dev-skip)      │
  │  phase-3 (Быстрая ходьба 1 мин)         │
  │  ↓ (auto после 1 мин)                   │
  │  result (модалка с детализацией)        │
  │  ↓ (клик ×)                             │
  └──── drilldown (теперь с заполн. Bayevsky)
       │
       ↓ (клик СТОП)
       claim (полноэкранный QR)
       │
       ↓ (клик «Готово»)
       → redirect на kiosk-v2.html
```

**Sub-state:** loss-of-connection — внутри `drilldown`, не отдельный state, только меняет чипы и замораживает live-виджеты.

**Реализация в HTML:**
- На `<body>` навешивается класс `state-empty / state-drilldown / state-phase-1 / state-phase-2 / state-phase-3 / state-result / state-claim`
- Все state-зависимые элементы (модалка, баннер empty, claim-экран, изменение Bayevsky-блока) показываются/скрываются через `body.state-X .element { display: ... }`
- Sub-state loss-of-connection — отдельный класс `state-disconnected` (можно комбинировать с drilldown), переключается toggle-кнопкой в dev-bar `[BLE: connected ⇄ потеря связи]`

## 14. Шкалы и формулы (для справки UX/UI агенту)

### 14.1. NSI (Neiry Stress Index) — отображается на live-виджете Stress

| Зона | Диапазон | Цвет | Подпись |
|---|---|---|---|
| Норма | 0-39 | success (зелёный) | `● норма` |
| Повышен | 40-59 | warning (амбра) | `● повышен` |
| Высокий | 60-100 | destructive (красный) | `● высокий` |

Семантика: чем выше NSI, тем выше стресс.

### 14.2. Индекс Баевского ИН — отображается на Bayevsky Results

Формула: `ИН = AMo / (2 · Mo · MxDMn)` (классический Институт медико-биологических проблем РАН).

| Зона | Диапазон ИН | Цвет | Вердикт |
|---|---|---|---|
| Истощение / низкая реактивность | < 30 | warning | 🟠 Слабая реактивность |
| Норма / здоровая реактивность | 30-150 | success | 🟢 Реактивность нормальная |
| Повышенная нагрузка | 150-500 | warning | 🟡 Повышенная реактивность |
| Стресс / срыв адаптации | > 500 | destructive | 🔴 Дисрегуляция |

Общий ИН Σ = медиана трёх фаз (Покой, Stroop, Ходьба).

### 14.3. Фильтр R-R интервалов (для NSI и Bayevsky)

Двухступенчатый:
1. **Медианный фильтр** ±20% от скользящей медианы
2. **Перцентильный** 5/95% от распределения сессии

Минимум 60 валидных R-R интервалов — иначе показывается `—` вместо числа.

**Замечание для UX/UI агента:** формулы не нужно реализовывать в wireframe. Mock-числа в §13 уже посчитаны корректно. Эти разделы — справка для понимания «откуда числа» и какие зоны окрашивать.

- [ ] Правая колонка содержит 4 виджета 2×2 + Bayevsky Results по §6

- [ ] Переключение URL `?id=kostya / ?id=alexey / ?id=guest` меняет mock-данные на всех виджетах
- [ ] URL без `&session=active` → state=empty с skeleton-плейсхолдерами и баннером
- [ ] URL с `&session=active` → state=drilldown с живыми данными для выбранного `id`

## 16. Что НЕ делать (анти-скоуп)

- НЕ делать рабочую игровую механику Stroop — только статический UI с 4 кнопками
- НЕ делать реальный QR-генератор для claim — placeholder/SVG-pattern достаточно
- НЕ делать реальный backend для Bayevsky — все ИН захардкожены в mock-данных
- НЕ ставить framework/build-step — только vanilla HTML/CSS/JS + CDN
- НЕ дублировать back-кнопку (только в dev-bar, не в top-bar)
- НЕ добавлять группы «УРОВЕНЬ СТРЕССА» и «КАЛИБРОВКА» в dev-bar (отличие от kiosk-v2)
- НЕ хардкодить размеры в инлайн-стилях (только Tailwind утилитарные классы)
