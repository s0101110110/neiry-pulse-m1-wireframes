# M2 Kiosk Wireframe — Design Spec
Дата: 2026-05-21 · Проект: Neiry Pulse · Milestone: M2 · Startup Village (25 мая) · **Срок: 2 часа**

---

## Контекст

На Startup Village нужен «вау-экран» дашборда для демонстрации экосистемы Neiry Pulse гостям стенда. Сценарий: оператор надевает браслет на гостя, на дашборде запускает «сессию гостя», 4–5 минут демонстрирует биометрию + предлагает стресс-тест на беговой дорожке. В конце — выдача QR-кода с PIN для получения полного отчёта в Telegram-боте.

Wireframe для согласования с командой (Кирилл — backend, Паша — frontend) до начала продакшен-вёрстки. Премиум-дизайн делается позже.

---

## Решения

| Параметр | Выбор | Обоснование |
|---|---|---|
| Аудитория экрана | Один UI на плазме (42") | Игнорируем риск shoulder-surfing QR; отчёт не содержит ПД |
| Сценарий | Оператор-управляемый | Сотрудник стенда подключает браслет, запускает сессию, после — RESET |
| Количество браслетов | Несколько физически, плазма фокусируется на одном | Переключение через dropdown в шапке |
| Идентификация гостя | Анонимно — по номеру браслета | `Neiry.Pulse #1`, без имён, без GDPR-рисков |
| Палитра | Фирменная Neiry (тёплый чёрный) | Единый бренд с M1, без передизайна |
| Файловая структура | Один HTML с переключателем состояний | Команда видит state machine за один таб |
| Стресс-тест | Без отдельного режима — реал-тайм реакция | Минимум скоупа, BPM/Calmness/IMU реагируют сами |

## Шапка

Фиксированная сверху, высота ~80px.

```
┌─────────────────────────────────────────────────────────────────────┐
│  Neiry Pulse    │  🟢 Подключено · Neiry.Pulse #1  │  [▼ #1]  02:15  │
└─────────────────────────────────────────────────────────────────────┘
   Fraunces            центр                        правая зона
```

| Зона | Содержимое |
|---|---|
| Левая | Логотип `Neiry Pulse` (Fraunces) |
| Центральная | `🟢 Подключено · Neiry.Pulse #1` (зелёная точка `#166534` + название активного браслета) |
| Правая | Dropdown переключатель браслета `[▼ #1]` + таймер сессии `02:15` (JetBrains Mono) |

Для wireframe — dropdown рисуем как **статичный** элемент в закрытом виде (`[▼ #1]`). Не интерактивный. Открытое состояние и переключение между браслетами — следующая итерация (после согласования базы с командой).

## Главная зона — Layout

Под шапкой — две колонки 60/40:

```
┌──────────────────────────────────┬──────────────────────────────┐
│                                  │                              │
│   ЛЕВАЯ КОЛОНКА — LIVE (60%)     │  ПРАВАЯ КОЛОНКА — FUTURE (40%) │
│                                  │                              │
└──────────────────────────────────┴──────────────────────────────┘
```

Правая колонка видна **всегда** (во всех состояниях) — это маркетинговая витрина для зевак в очереди.

## Состояния (State Machine)

### Состояние IDLE — нет активного гостя

**Левая колонка:**
- По центру колонки крупный текст: `Подключите Neiry.Pulse и нажмите СТАРТ` (вторичный цвет `#7a766d`)
- Большая кнопка `[ СТАРТ ]` (accent `#831843`, Manrope Bold)
- Нет графиков, нет метрик

**Правая колонка:** показывается с фейк-данными (см. ниже)

---

### Состояние ACTIVE — идёт сессия

**Левая колонка (сверху вниз):**

1. **Live BPM** — крупная цифра `84`, JetBrains Mono, ~120px, цвет `#831843`
   - Рядом — пульсирующая иконка сердца (CSS-анимация `@keyframes pulse`)
   - Подпись `уд/мин` (вторичный цвет)

2. **Бегущий график пульса** — SVG line chart, высота ~120px
   - Стилизованный, плавно бегущий вправо
   - Реализация: JavaScript-обновление SVG-path раз в секунду из массива fake BPM

3. **Neiry Stress / Calmness Score** — горизонтальный градиентный бар
   - Заголовок: `Stress / Calmness Score` (Manrope)
   - Бар: 100% ширины колонки, высота ~24px, градиент слева направо: красный (стресс) → жёлтый → зелёный (спокойствие)
   - Подписи под концами бара: слева `СТРЕСС`, справа `СПОКОЙСТВИЕ` (мелко, вторичный цвет)
   - Ползунок (вертикальная белая полоска) на позиции 68% от левого края (= зелёная зона = спокойствие)
   - Подпись под баром: текущее значение `68 · Спокойствие` (JetBrains Mono) и пояснение `Реал-тайм оценка вегетативного статуса` (вторичный цвет, мелко)

4. **IMU статус** — мелкий индикатор
   - В wireframe фиксируем значение: `IMU: 🟢 Покой`
   - JetBrains Mono, вторичный цвет
   - (Состояние `🟡 Активность` для будущей итерации)

5. **ЭКГ-кнопка** — disabled
   - `[ 🔒  ЭКГ-сканирование ]`
   - Фон `#1a1814`, текст `#4a463e` (placeholder-цвет)
   - НЕ кликабельна, никаких tooltip про VANTA/M3

6. **Главная кнопка управления (низ колонки):**
   - `[ СТОП ]` — accent `#831843`

---

**Левая колонка:** превращается в карточку отчёта.

```

│              СЕССИЯ ЗАВЕРШЕНА                │
│                                              │
│   [мини-график BPM за всю сессию, SVG]       │
│                                              │
│   ┌─────────────────┐                        │

│   │ Средний Calmness    68                   │

│                                              │
│                              [ СБРОСИТЬ ]    │

- Заголовок `СЕССИЯ ЗАВЕРШЕНА` — Manrope Bold, крупный, по центру
- Мини-график BPM за сессию (SVG line, фейк-данные)
- Табличка с 3 числами (JetBrains Mono)
- Подзаголовок `Полный отчёт — в Telegram-боте` (вторичный цвет)
- Кнопка `[ СБРОСИТЬ ]` справа внизу (accent `#1e3a8a`) — переводит обратно в IDLE

**Поверх — модальное окно QR (фиксированное, центр экрана):**

```
┌────────────────────────────────────────┐
│                                        │
│   📱 Заберите отчёт в Telegram         │

- PIN — крупно (JetBrains Mono, ~48px, accent `#831843`)
- Кнопка `[ ЗАКРЫТЬ ]` (accent `#1e3a8a`) — закрывает модалку, но левая колонка остаётся в состоянии PENDING_CLAIM (для wireframe — кнопка просто скрывает модалку)

## Правая колонка — FUTURE / Витрина платформы

Видна во всех состояниях. Содержит 4 виджета с фейк-данными.

**Плашка-заголовок** (поверх колонки, не скроллится):
```
[ ПРЕДИКТИВНАЯ АНАЛИТИКА · ТРЕБУЕТСЯ 14+ ДНЕЙ НОШЕНИЯ ]
```
- Полупрозрачный фон (`bg-[#1a1814]/80`)
- Border `#2a2620`
- Текст `#7a766d`, JetBrains Mono uppercase, ~12px
- Padding ~12px по вертикали

**4 виджета (сверху вниз):**

### Виджет 1 · Neiry Readiness
- Заголовок: `Neiry Readiness`
- Donut chart (SVG), процент по центру: `82%`
- Подзаголовок: `Индекс восстановления`
- Подпись внизу: вторичный текст, например `На основе сна, HRV и активности за 14 дней`

### Виджет 2 · Качество сна
- Заголовок: `Качество сна`
- 3 цифры с подписями: `Deep 21%` / `REM 24%` / `Light 55%`
- Горизонтальная stacked-bar (3 сегмента) показывает пропорции под цифрами
- Подпись внизу: `Last night · 7ч 42мин`

### Виджет 3 · Burnout Risk
- Заголовок: `Burnout Risk`
- Крупный статус: `🟢 Низкий риск`
- Подзаголовок: вторичный текст `Скользящее среднее стресса за 14 дней`
- (Без упоминания B2B-кейсов)

### Виджет 4 · Drowsiness Predictor
- Заголовок: `Drowsiness Predictor`
- Статус: `🟢 Активный режим`
- Подпись: `Алерт за 5–10 мин до микросна`
- (Без упоминания B2B-кейсов)

Все виджеты — фон `#1a1814`, border `#2a2620`, padding ~16px, скруглённые углы (`rounded-lg`).

## Анимации

**В скоупе (минимум):**
- CSS-анимация `@keyframes pulse` на иконке сердца в ACTIVE (масштабирование 1.0 → 1.1 → 1.0, 1s loop)
- Smooth transition (`transition-all duration-300`) при смене состояний (opacity/transform)
- Fake live BPM: JavaScript обновляет цифру каждые 800–1200 мс случайным значением в диапазоне 76–92
- Fake график: SVG line обновляется в такт с BPM

**НЕ в скоупе:**
- Реалистичные графики через Chart.js / D3
- Стресс-тест с анимированной кардиограммой
- Анимация открытия/закрытия модалки (стандартный show/hide)

---

## Скоуп (срок 2 часа)

**Входит:**
- Один HTML файл `wireframes/m2/kiosk.html`
- Tailwind CSS через CDN
- 3 состояния через toggle (служебная панель сверху)
- Fake live BPM через setInterval
- Минимум CSS-анимаций (pulse сердца, transitions)
- Все виджеты с фейк-данными
- Модалка QR (плейсхолдер QR — статичное SVG)

**НЕ входит:**
- Премиум-визуал, реалистичные графики
- Реальный API / WebSocket
- Переключение между N браслетами в UI (всегда `Neiry.Pulse #1`)
- Отдельный режим стресс-теста
- Адаптивность под другие размеры (только 1920×1080)
- Реальный QR с Telegram Deep Link
- Анимация перехода между состояниями (только opacity)
