# Bayevsky Test Block Redesign — kiosk-drilldown.html

> **Дата:** 2026-05-28
> **Целевой файл:** `docs_web/wireframes/m2/kiosk-drilldown.html`
> **Скоуп:** Только нижний блок «Тест Баевского» в правой панели. Остальное не трогаем.

## 1. Контекст

PM решил упростить визуал теста Баевского до одной фазы (3 минуты неподвижности) и заменить «технический» результат (ИН + текст вердикта) на образный — 5 архетипов-«стресс аватаров» (Дзен-мастер / Самурай / Гепард / Берсерк / Вулкан). Формула расчёта SI (`AMo / (2·Mo·MxDMn)`) и одноступенчатый медианный фильтр R-R 20% не меняются.

## 2. State machine + dev-bar

### Текущий dev-bar (СЦЕНАРИЙ-группа, 8 pills)
`Empty · Enter-Name · Drilldown · Phase-1 · Phase-2 · Phase-3 · Result · Claim`

### Изменения
- **Удалить:** `Phase-1`, `Phase-2`, `Phase-3`
- **Добавить:** `test-start`, `test-end`
- **Сохранить:** `Empty`, `Enter-Name`, `Drilldown`, `Result`, `Claim`

### Финал — 7 pills
`Empty · Enter-Name · Drilldown · test-start · test-end · Result · Claim`

### Семантика состояний (только содержимое блока Баевского)

| Состояние | Содержимое блока |
|---|---|
| `Empty` | Пустое кольцо + по центру текст «**Какой твой стресс аватар?**» (CTA-копирайт) |
| `Enter-Name` | Без изменений (текущая логика) |
| `Drilldown` | Пустое кольцо + подпись «Готов к тесту», кнопка «Тест» enabled при IMU=rest |
| `test-start` | Кольцо countdown 3:00, заполняется wine #831843, текст «Стой неподвижно, дыши спокойно» |
| `test-end` | Кольцо 100% wine, по центру подпись «ТЕСТ ЗАВЕРШЁН» (transition, ~1.5 сек, далее auto → `result`) |
| `result` | Hero-карточка matched-уровня в горизонтальном layout (см. §5) |
| `claim` | QR-код (без изменений) |

## 3. Кольцо-таймер (`test-start`)

- SVG ring, заполняется по часовой от 12 ч
- Stroke wine #831843, dashoffset 100% → 0% по мере убывания таймера
- Внутри только цифровой таймер `mm:ss` (font-data, t-display)
- Под кольцом: `Стой неподвижно, дыши спокойно` (t-body, muted-fg)
- Без breathing widget
- Старт мгновенный по клику «Тест» (без pre-roll)
- СТОП → возврат к `Drilldown`, данные не сохраняются
- На 0:00 → silent auto-переход в `test-end`

## 4. Transition state (`test-end`)

Кольцо застывает в 100% wine, цифра таймера скрывается, по центру кольца появляется надпись `ТЕСТ ЗАВЕРШЁН` (font-display, t-caption, uppercase, letter-spacing). Длится ~1.5 сек, затем auto → `Result`. В dev-bar pill `test-end` позволяет тестеру задержаться на этом стейте для инспекции.

## 5. Result-карточка — горизонтальный layout

Блок Баевского — широкий и узкий по высоте. Карточка занимает всю ширину блока, layout горизонтальный.

```
┌──────────────────────────────────────────────────────────────────┐
│  ШКАЛА · СТРЕСС АВАТАР                                (t-eyebrow)│
│  ┌──────┐                                                        │
│  │ 80×80│  ГЕПАРД              Симпатическая мобилизация,        │
│  │ img  │  ИН 187              продуктивный фокус                │
│  └──────┘  (color-accent)      (t-body, fg)                      │
└──────────────────────────────────────────────────────────────────┘
   ↑img      ↑col-name+ИН        ↑col-описание (flex: 1)
   80×80px   мин-ширина 140px    оставшаяся ширина
```

- **CSS grid:** `grid-template-columns: 80px auto 1fr; gap: 24px; align-items: center;`
- **Колонка 1:** `<img>` 80×80 px (PNG из `assets/levels/`)
- **Колонка 2:** имя уровня (t-h3, font-display, color-accent) + строкой ниже `ИН {n}` (t-data, JetBrains Mono, color-accent)
- **Колонка 3:** описание состояния (t-body, fg, white-space normal)
- **Без кнопки RESET**
- **Семантика цвета (color-accent):** Дзен/Самурай → `--ok`, Гепард → `--info`, Берсерк → `--warn`, Вулкан → `--danger`. Применяется к имени + бейджу ИН. Фон карточки — `--card`, без заливки
