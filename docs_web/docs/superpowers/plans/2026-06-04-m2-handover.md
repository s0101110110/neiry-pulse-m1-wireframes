```markdown
# Метрики HRV и Steps — спецификация M2

## HRV (Heart Rate Variability)

### Источник данных
- Veepoo SDK call: `readRRIntervalByDay(date)` → массив R-R интервалов (мс)
- Backend агрегация: `POST /v1/metrics/hrv/calculate`
- Окно расчёта: 3-5 минут (для slow Баевский ИН) или весь день (для daily summary)

### Формулы
- **RMSSD** (мс) = √(Σ(R-R[i+1] − R-R[i])² / (N−1))
  - Norm range: 20-100 мс (взрослый, в покое)
  - Используем: основной показатель «daily HRV»
- **SDNN** (мс) = √(Σ(R-R[i] − mean)² / N)
  - Norm range: 30-150 мс
  - Используем: контекст для RMSSD
- **Баевский ИН** (Stress Index Slow) = AMo / (2·Mo·MxDMn)
  - Используется в TG-отчёте после сессии (не в live)
  - Фильтр R-R-артефактов: 5% перцентильный

### Пороги (для UI-визуализации, clinical-семантика)
- 🟢 Норма: RMSSD > 50 мс
- 🟡 Сниженная: RMSSD 30-50 мс
- 🔴 Низкая: RMSSD < 30 мс (риск перетренированности / стресса)

### Визуализация
- **Dashboard (`dashboard-corporate.html`)**: KPI card «HRV» с числом RMSSD в мс + sparkline 7 дней
- **Mobile (`mobile-main.html`)**: блок «Сегодня HRV» с числом + цветом-индикатором (точка 8×8)
- **TG-отчёт**: Баевский ИН (число) + вердикт текстом

---

## Steps (шаги)

### Источник данных
- Veepoo SDK call: `readDailyStepByDay(date)` → integer (шаги за день)
- Backend агрегация: `GET /v1/metrics/steps/daily?user_id=X&from=YYYY-MM-DD`

### Формула
- Сырые данные SDK: integer шагов за день
- Активные минуты: расчёт на backend через cadence (если cadence > 100 шагов/мин в 1-мин окне → активная минута)

### Пороги
- Daily target: 10 000 шагов (универсально, без персонализации в M2)
- Active minutes target: 30 мин/день (WHO)

### Визуализация
- **Dashboard**: KPI card «Шаги сегодня» = «3 200 / 10 000»; bar progress
- **Mobile**: круговой прогресс с числом в центре

---

## Зависимости
- Расчёты HRV — на backend (slow, окно 3-5 мин); SDK даёт только R-R массив
- Steps — на SDK (fast), backend только хранит и агрегирует
- Все эндпоинты в M2-скоупе (см. `milestones/m2.md`)
```

```markdown
# Demo Script — «Спортсмен» (M2)

**Длительность:** 3-5 минут на гостя
**Площадка:** Startup Village 28-29.05.2026
**Команда:** Костя (PM/перформер) + Алексей (B2B)
**Оборудование:** 3 браслета Base, плазма 42", admin-ноут с drill-down, беговая дорожка

## Цель демо
Показать инвестору/корпорату, что Neiry умеет ловить **физиологический стресс в реальном времени** и визуализировать его понятным образом (NSI 0-100, светофор).

## Персона гостя
- Корпорат HR / инвестор / журналист — без технического бэкграунда
- Хочет понять «что это и зачем», за 3 минуты, на пальцах

## Сценарий (5 актов)

### Акт 1: Hook (30 сек)
- Жест на плазму: «Вот два наших коллеги, прямо сейчас, в работе.»
- Показать живые карточки 2 анкеров (Костя + Алексей) с пульсом и NSI
- «Зелёный — норма, жёлтый — повышен, красный — острый стресс»

### Акт 2: Hands-on (60 сек)
- «Хотите попробовать? Наденем браслет, 12 секунд калибровки.»
- Браслет → mobile-pairing → ввод имени → calibrating → active
- Карточка гостя появляется на плазме третьей

### Акт 3: Wow (90 сек)
- Drill-down на карточку гостя (тап на admin-ноуте)
- Showcase: BPM hero, NSI gauge, Stress card, Activity (IMU)
- **Провокация:** «Сделайте 15 приседаний / пройдите Stroop test на планшете»
- Метрики гостя меняются в реальном времени — NSI ползёт в жёлтую/красную зону

### Акт 4: Деpth (60 сек, опционально)
- «Хотите увидеть детальный анализ?» — кнопка «Тест Баевский»
- 3 фазы: покой (3 мин) → Stroop → беговая дорожка (5 мин)
- После — экран результата с ИН и вердиктом

### Акт 5: Close (30 сек)
- «Это будет работать для всей вашей команды. Дашборд HR здесь.» — показать `dashboard-corporate.html`
- «Пилот, 50+ сотрудников, 3 месяца. Контакт?»
- QR-визитка / Telegram

## Fallback-сценарии

| Что сломалось | Что делать |
|---|---|
| Браслет не подключается за 30 сек | «Сейчас не подключается, это случается на стенде. Покажу на нашем коллеге» — drill-down на Костю |
| Гость передумал надевать | Сразу к Акту 3 на анкере Кости (он работает, его NSI меняется когда он шутит / делает приседания) |
| Плазма зависла | На admin-ноут, dashboard-corporate как backup |
| Беговая дорожка занята | Только Stroop test для Wow-провокации |

## Запретные темы (NDA)
- НЕ называть конкретных B2B-клиентов
- НЕ показывать VITRO/VANTA/VIGOR
- НЕ обещать ECG-сертификацию

## Что говорим про железо
- «Браслет Base — то, что на руке сейчас»
- «Health и Sport (с GPS и ECG) — начало июня, отдельная линейка»
- НЕ позиционировать Base как final-продукт
```
