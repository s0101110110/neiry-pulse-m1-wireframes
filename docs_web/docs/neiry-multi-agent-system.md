# Мульти-агентная система Neiry Pulse

> Документ для разработчика: как устроена система AI-агентов в проекте, и чем она отличается от встроенных субагентов Claude Code.

---

## Что такое встроенные субагенты Claude Code

Claude Code имеет встроенный инструмент `Agent tool` — он запускает **временного** агента для одной задачи.

```
Claude (главный) → вызывает Agent tool → агент выполняет задачу → возвращает результат → агент исчезает
```

**Свойства встроенных субагентов:**
- Живут только во время одного вызова
- Не помнят прошлых сессий
- Не накапливают знания
- Не имеют своей «личности» или специализации
- Не общаются между собой — только через главный агент

Это хорошо для изолированных задач: «прочитай эти 10 файлов и скажи мне X». Но не подходит для долгосрочной командной работы.

---

## Что мы построили: постоянная команда агентов

Вместо временных субагентов — **постоянная команда из 4 агентов**, каждый из которых:

- Живёт в своей папке
- Имеет свой `CLAUDE.md` — это его «личность», роль, правила поведения
- Накапливает знания в `lessons-learned.md` между сессиями
- Получает задачи через `BRIEF.md` от PM

```
PM (docs_web/) ──────────────────────────────────────────────────────────────
     │                                                                       │
     ├──→ UX/UI Дизайнер (UI_assets/)    ├──→ Frontend Dev (dashboard/)      │
     │         CLAUDE.md                 │         CLAUDE.md                 │
     │         BRIEF.md                  │         BRIEF.md                  │
     │         lessons-learned.md        │         lessons-learned.md        │
     │                                   │                                   │
     └──→ R&D Researcher (research/)     └──→ knowledge-base/ (общая память) │
               CLAUDE.md                           PROJECT_STATE.md          │
               BRIEF.md                            tasks/, decisions/        │
               lessons-learned.md                  open-questions/           │
               metrics/, sdk/                      people/                   │
```

## Ключевые механизмы

### 1. CLAUDE.md — «должностная инструкция» агента

Каждый агент при старте сессии читает свой `CLAUDE.md`. Там написано:
- Кто он (роль, специализация)
- Что делать в начале каждой сессии (что читать, что проверить)
- Какой уровень автономии (что делать самостоятельно, что согласовывать)
- Как передавать результаты

Пример (`research/CLAUDE.md`):
```markdown
## Identity
Я — R&D Researcher. Изучаю железо браслетов, SDK, метрики здоровья.
Объясняю сложные технические вещи простым языком.

## Session Start
1. Прочитать INDEX.md (что уже изучено)
2. Прочитать lessons-learned.md (прошлые ошибки)
3. Прочитать BRIEF.md (задание от PM)
```

**Чем отличается от встроенного субагента:** встроенный агент получает роль только в промпте вызова и забывает её сразу. Этот агент «помнит» свою роль всегда — она записана в файле.

---

### 2. BRIEF.md — канал делегирования

PM пишет задачу в `BRIEF.md` нужного агента. Агент читает его при старте.

```
PM пишет:
  research/BRIEF.md → "Обработай Excel с метриками → создай vitro-metrics-registry.md"

R&D Researcher читает BRIEF.md → выполняет → результат в research/metrics/
PM читает результат → очищает BRIEF.md
```

**Чем отличается:** встроенный субагент получает задачу в момент вызова и выполняет её немедленно. BRIEF.md позволяет **асинхронно** ставить задачи — агент прочитает их когда его запустят.

---

### 3. lessons-learned.md — самообучение

Каждый агент ведёт журнал ошибок и правил. При старте следующей сессии агент читает его — и не повторяет прошлых ошибок.

```markdown
## 2026-05-16 — WebSocket reconnect
Проблема: браузер закрывает WS при потере фокуса вкладки.
Правило: всегда реализовывать exponential backoff reconnect. Уже реализовано — не удалять.
```

**Чем отличается:** встроенный субагент каждый раз начинает с нуля — без памяти о прошлых решениях. Наш агент накапливает опыт между сессиями.

---

### 4. Knowledge-base — общая память команды

Отдельный git-репозиторий `/knowledge-base/` — это «мозг» всей команды:

```
knowledge-base/
├── PROJECT_STATE.md      ← живой снимок: что сделано, что в работе, дедлайны
├── tasks/
│   ├── konstantin.md     ← задачи по людям
│   ├── kirill.md
│   └── nikita.md
├── decisions/
│   └── 2026-05.md        ← зафиксированные решения (HRV в M3, iOS в M2...)
├── open-questions/
│   └── index.md          ← нерешённые вопросы
├── milestones/
│   ├── m1.md, m2.md      ← скоуп и статус милстоунов
├── people/index.md       ← кто есть кто в проекте
└── INBOX/
    ├── meetings/          ← транскрипции встреч (обрабатываются PM)
    └── raw/               ← сырые файлы (Excel, PDF) для агентов
```

**Чем отличается:** встроенный субагент читает только то, что ему передали в промпте. Наши агенты имеют доступ к единому источнику правды, который обновляется каждую сессию.

---

### 5. Session Lock — защита от потери данных

При старте сессии создаётся файл `.session-lock`. При завершении:
1. Обновляется `PROJECT_STATE.md`
2. Удаляется `.session-lock`
3. Делается git commit в knowledge-base

Если `.session-lock` существует при следующем старте — значит прошлая сессия завершилась аварийно и нужно проверить что потеряли.

**Stop Hook** в `~/.claude/settings.json` показывает напоминание в конце каждой сессии:
```json
"Stop": [{
  "hooks": [{
    "type": "command",
    "command": "echo '{\"systemMessage\": \"⚠️ Обнови PROJECT_STATE.md, удали .session-lock, закоммить\"}'",
    "async": false
  }]
}]
```

## Сравнительная таблица

| Характеристика | Встроенные субагенты Claude | Наша система |
|---|---|---|
| Время жизни | Один вызов | Постоянные, между сессиями |
| Память | Только контекст вызова | Файлы: CLAUDE.md, lessons-learned, KB |
| Специализация | Определяется промптом | Зафиксирована в CLAUDE.md |
| Самообучение | Нет | Да — через lessons-learned.md |
| Делегирование | Немедленное (синхронное) | Асинхронное через BRIEF.md |
| Общая память | Нет | knowledge-base/ git-репо |
| Стоимость | Токены за каждый вызов | Токены только при запуске агента |
| Сложность | Низкая (встроено в Claude) | Средняя (папки + файлы + соглашения) |
| Подходит для | Разовых изолированных задач | Долгосрочной командной работы |

## Как это работает на практике (пример)

**Задача:** обработать Excel с метриками браслета

**Со встроенным субагентом (разово):**
```
PM → Agent tool: "прочитай Excel, скажи что там"
→ агент читает → возвращает текст → забывает
```
В следующий раз придётся объяснять заново что такое браслет, что такое HRV, какой формат нужен.

**С нашей системой (постоянно):**
```
PM → пишет в research/BRIEF.md задание
→ запускает R&D Researcher агент
→ агент читает CLAUDE.md (знает свою роль)
→ агент читает INDEX.md (знает что уже изучено)
→ агент читает lessons-learned.md (знает прошлые ошибки)
→ агент читает BRIEF.md (знает конкретное задание)
→ выполняет → сохраняет в research/metrics/
→ PM очищает BRIEF.md, обновляет PROJECT_STATE.md
```
В следующий раз агент уже знает контекст проекта — не нужно объяснять с нуля.

## Технические требования

- Claude Code (CLI) — основной интерфейс
- Git — для knowledge-base и основного репо
- Папочная структура — единственная «инфраструктура»
- Никаких серверов, баз данных, API

**Это чистая архитектура на файлах.** Вся «магия» — в том, что Claude Code при запуске читает `CLAUDE.md` из рабочей директории и следует инструкциям.
