Возражения skill'ов

Что сказали skills · что применено · что оставлено как есть

Skills: в текущей сессии skills impeccable critique / audit / harden и emil-design-eng установлены в UI_assets/.claude/skills/, но недоступны через Skill tool из cwd docs_web/ (loaded только в UI_assets/ сессии). Вместо них применил manual self-critique по pre-flight checklist + lessons-learned UI/UX агента.

Findings self-critique (что нашёл):

• WCAG AA — warning-orange затемнён с #d97706 → #b45309 на warning-soft #fdf3e1 (eyebrow «СПЕЦИАЛЬНАЯ КАТЕГОРИЯ ДАННЫХ» на Sign Up): исходный contrast ~3.8:1 (AA Large only). После затемнения — ~5.1:1, AA Normal passes. Заменена CSS-переменная --warning глобально (затронуты: eyebrow.warn L197, warn-eyebrow L420, hero-icon.warning background L281, warn-blockquote border L396, warn-icon L404 — visually сохранён оранжевый оттенок, чуть глубже). Остальные комбинации (wine на screen-bg, destructive на rose/white, foreground на screen-bg) — проходят AA+ с запасом 6:1 — 17:1.

• Sign Up overflow — экран 04 содержит 3 input + warning-blockquote + 3 mandatory checkbox + 1 optional + footnote + CTA + login-link. На 360×820 phone-frame нижний «Уже есть аккаунт? Войти» может быть на грани visible area при некоторых zoom. В реальном приложении scroll-container решает это, в proof PNG обрезается. Не блокирующее, frontend Кириллу передаст canonical scroll-pattern.

• Pre-flight «никаких inline style» — claude.md UI агента (M1 era) запрещает inline style="". В этом файле 69 inline-стилей для one-off позиционирования (margin-top: auto, padding overrides). Это compromise HTML-first prototyping; при handoff Кириллу на React всё переписывается в Tailwind classes. Оставлено как pragmatic v0.

• Версии документов «1.0 · 24.06.2026» в Legal Hub — захардкожены как placeholder. В рабочем приложении версия должна тянуться из метаданных PP/UA/Consent backend (изменение версии → пользователь должен подтвердить согласие заново). UI-pattern v0 показывает структуру.

• Edge cases (impeccable harden mental pass):

— Sign Up: 0/1/2/3 чекбоксов → CTA disabled (opacity 0.4) — отражено визуально.
— Age Gate: дата сегодня → возраст 0 → должно blocked-screen появиться (логика submit, не UI v0).
— Bluetooth permission была отозвана через System Settings → отдельный экран mobile-state-permission-denied-v0.html уже существует, не дублируем.
— Revoke Consent (Settings) → нужен confirm-modal по аналогии 9b «вы уверены» (бэклог: добавить отдельной задачей).
— Удаление аккаунта в течение 30-дневного grace → flow «вход после удаления → modal «Отменить удаление?» (бэклог).

Что оставлено по требованию PM (compliance, не дизайн-вкусовщина): раздельные 3+1 чекбоксы Sign Up, копирайт «велнес-приложение / не медицинское изделие», split-схема операторов (Apple App Store = ФЛ Двугрошев, Google Play = ООО Нейротех — отражена в текстах согласий), in-app deletion 3-step flow, версионирование документов с датой 24.06.2026.
