Возражения skill'ов — на разбор PM

v2 собрана формальным прогоном skills: redesign-existing-projects → impeccable typeset → impeccable colorize → impeccable harden → impeccable polish. Сверка велась с 6 живыми снимками Bevel в screenshots/references/ (Home / HRV / Stress / Sleep / Recovery / Onboarding). Замкнутые константы (тёплый крем · wine single-accent · Golos+JetBrains · light-mode) оставлены даже там, где skill или живой Bevel предлагали иначе — по явной директиве PM. Ниже — что Bevel делает иначе и почему мы отклонили или отложили. Пересмотр PM 26.06 (v3): крем → прохладно-серый и single-accent → мультисемантика приняты. Шрифт остаётся наш (Golos Text + Pulse Mono), Dark Mode — на Sleep-разделе. Обновлённые вердикты отмечены ниже.

impeccable typeset Перевести все font-size с px на rem (NEVER-правило: уважать системный zoom) «px для font-size игнорирует пользовательский масштаб шрифта — это провал доступности» AMEND — px осознанно для mockup Это device-mockup с фиксированным 360px-экраном в bezel: px даёт пиксельную точность пропорций (Material/Polaris фиксируют app-UI так же). rem-перевод — при переносе в продакшн-React (Кирилл), где вьюпорт резиновый. В прототипе вычищены полушаги: 11.5/12.5/13.5 → 12/13/14, убран неиспользуемый вес 900
impeccable colorize «Добавить стратегическую палитру» — команда colorize по определению про обогащение цветом «Single-accent монохром — это “timid grayscale”, который colorize и призван лечить» AMEND — применён как contrast-hardening Для product-register colorize сам предписывает Restrained + semantic-first — это и есть наша модель. Поэтому colorize применён не как «новые hue», а как WCAG AA: ink-dim #9C968E→#726C62 (2.7→5.2:1), amber-text #E89500→#B45309 (2.4→5.0:1), green-verdict #00A653→#027038 (3.2→6.2:1), banner-icon #C58A1F→#7E5B17. Bio-severity (green/amber/red/teal) сохранена точечно как семантика состояния
impeccable colorize Добавить 2-3 «mood»-цвета (sage, ochre, lavender) для секций recovery/strain/sleep «Bevel использует один warm accent + multi-tonal pastels на category cards. Single-accent = монотонно» REJECT — нарушает single-accent правило Сохранил wine + tonal cream-ladder. Differentiation через layout/icon-shape, не через дополнительные hue. Bio-палитра --bio-low/mid/high — точечно на severity, не decorative
impeccable colorize Banner «Демо-режим» сделать wine, а не amber «Amber на cream-bg перегружает warm-spectrum, single-accent должно держать монохромию» AMEND — оставил amber Amber на Демо-плашке = warning-семантика (это инфо-предупреждение про fake data). Если делать wine — спутается с brand-action. Оставил amber, но сделал tone-down (border #F5E2C0, text #7E5B17)
impeccable harden Добавить полные empty / loading / error экраны, per-element ARIA-labels, focusable nav-табы «Product-register требует default/hover/focus/loading/error на каждом компоненте + SR-разметку» DEFER — production-scope Это статический визуальный пруф 3 экранов, не shipping-app. В пруфе сделано системно: глобальный reduced-motion, :focus-visible ring, overflow-защита (device-ID / e-mail / i18n). Полные state-экраны + per-element ARIA + клавиатурные nav-табы — при переносе в React (Кирилл). «Демо-режим»-banner уже демонстрирует state-паттерн

В русском длинное тире обязательно («Мама 67 лет», «для Здоровья — Тренер и Мама»). Орфографическая корректность RU (директива пользователя) выше англо-правила skill
