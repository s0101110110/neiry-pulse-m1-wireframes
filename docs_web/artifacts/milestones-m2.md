## Сценарий стенда (финал, мысли PM 24.05)
- **3 браслета на стенде:** 2 анкера (постоянные демки, дают визуальный фон активности) + 1 для гостя.
- **Калибровка гостя 12 сек** (расширение до 16 при недостатке семплов) — спокойный baseline, затем плазма стартует с NSI.
- **Drill-down с карточки** — тап → детальный экран метрик гостя.
- **Опциональная кнопка «Тест Баевский»** на drill-down: 3 мин неподвижно + Stroop test + беговая дорожка (~5 мин) → запускает slow-расчёт ИН.
- **Fake live BPM не нужен** — анкеры дают реальный фон, гость — реальный baseline.
- **Линейка устройств на стенде:** только **Base** (Health/Sport приедут начало июня). В питче не позиционировать Base как final-продукт.

## Stress Index на стенде (Fast/Slow split)
- **Fast UI (плазма):** **NSI (Neiry Stress Index)** = `100 − (Current_BPM − Baseline_BPM) × 2`, clamp [0..100]. Семантика: ↑ значение = ↑ стресс. Зоны: 🟢 0–39 НОРМА / 🟡 40–59 ПОВЫШЕН / 🔴 60–100 ВЫСОКИЙ. Wow-эффект.
- **Slow Backend (TG-отчёт после сессии):** **Баевский ИН** = `AMo / (2·Mo·MxDMn)`, окно 3-5 мин, фильтр R-R-артефактов 5%.

## Стратегия стенда (Product Sync 16.05)

### Концепция «Медицинский светофор»
Гость не понимает, что значит «HRV 45мс». Нужен композитный индекс **0–100** с цветовой зоной (NSI, семантика ↑ = стресс ↑):
- 🟢 **0–39 НОРМА** — пульс ровный, ВСР стабилен
- 🟡 **40–59 ПОВЫШЕН** — пульс ускоряется, лёгкое напряжение
- 🔴 **60–100 ВЫСОКИЙ** — острый стресс, резкие скачки пульса

### Short-list метрик для плазмы (Kiosk Mode)
1. **Live Пульс (BPM)** — огромная цифра с бьющимся сердцем, обновление 1Гц
2. **Кислород (SpO2 / CO2 proxy)** — статичная цифра (98%) для успокоения
3. **NSI · Neiry Stress Index (0–100)** — главная метрика, светофор (↑ = стресс ↑)

### Wow-сценарии (нужно спровоцировать реакцию за 3 минуты)
- **Когнитивный стресс (B2B):** тест Струпа 60 секунд на планшете → индекс уходит в жёлтую зону
- **Физический стресс (B2C):** 15–20 приседаний → переход в «вторую жиросжигающую зону»

### Что НЕ показываем (защита от возражений)
- ❌ Артериальное давление — физически недостоверно без манжеты, юридический риск
- ❌ Глюкоза — невозможно измерить оптикой
- ❌ Состав крови (липиды, мочевая) — чистый маркетинг Veepoo

## Flow стенда (PIN + Deep Link)
Из критики flow Кирилла (`INBOX/raw/_Логика Телеграмм бота v2.docx`):

### State Machine карточки браслета
1. **IDLE** — пустая карточка, кнопка [СТАРТ]
2. **ACTIVE** — огромный пульс, ползёт график, кнопка [СТОП]
3. **PENDING_CLAIM** — на админ-ноуте QR (deep link) + PIN, кнопка [ЗАКРЫТЬ (RESET)]

### Контракт PIN
- Бэкенд: на `POST /sessions/stop` генерит 4-значный PIN, кладёт в Redis на 24ч, возвращает фронту
- Фронт: генерит QR с `https://t.me/NeiryPulseBot?start=PIN`. **QR показывается только на админ-ноуте**, не на плазме (приватность!)
- TG-бот: ловит `/start PIN` через deep link или ручной ввод → отдаёт отчёт → сжигает PIN
- RESET на фронте чистит карточку, но **PIN на бэкенде живёт 24ч** — гость может ввести позже

### Приватность стенда
- Данные не сохраняются между гостями автоматически — нужен PIN
- Дезинфекция браслетов между участниками
- На плазме после STOP — «СЕССИЯ ЗАВЕРШЕНА», без QR
