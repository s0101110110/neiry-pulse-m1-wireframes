§1 · Хронология стенда

Технические инциденты

12 заметных сбоев за два дня. Большинство сходится в 3 первопричины: нестабильная Wi-Fi-сеть стенда, BLE-стек VITRO (известный bug поставщика — see §2 vendor) и отсутствие release-checklist (старая APK на демо-устройстве).

Топ-факапы

BLE: три браслета одновременно не парятся. Подключение получалось «раза с 10-го» через строго последовательное парирование пар «телефон+браслет». Root cause подтверждён поставщиком 03.06: «Two phones switch the connection because the Bluetooth connection is not manually disconnected» (Vigor bug 1).

28.05 16:01 · Костя «три браслета одновременно не парятся» · 22:09 Алексей разбор

Wi-Fi congestion: телеметрия теряется. На пике посещаемости запросы из mobile app пропадали, данные не доходили до бэка. PM 14:04: «Не уходят данные на бэк. Нужно правда делать реинжиниринг». Кирилл подтвердил: «запросы из-за количества людей в сети теряются». Offline-буфера / retry-логики у клиента нет.

28.05 13:38–16:11 · Костя × Кирилл

Android foreground service: в коде, не в APK. PM требовал «пульс на закрытом экране», Кирилл «я сделал», но на устройстве Алексея сборка была старая. Классический release-checklist fail.

28.05 14:35–14:59 · Костя × Кирилл

Yandex Cloud упал, personas статика недоступна. Второй день стенда: «нет иконки у героя» (Никита 10:35) → «вообще всё сломалось» (Алексей 10:37). Кирилл уточнил, что упала статика хостинга, не бэк. Fallback-площадки нет.

29.05 10:35–10:39 · повтор 02.06 14:43

NSI = 0 на якоре Алексея. Метрика стресса показывала ноль на демо-якоре. Технически объяснимо: пульс 80 при низкой baseline → формула давала 0. Но визуально читалось как «сломано». Формула в changelog Кирилла исправлена (28.05 → 02.06): инверсия знака, теперь корректно растёт со стрессом.

§2 · Чиним и усиливаем

Action items M3

18 задач: 12 done 2 в работе 4 vendor. «Done» закрыты Кириллом и Пашей за 28.05 → 03.06, консолидация v1+v2 kiosk-фронта подтверждена визуальной проверкой production-домена 04.06.

✓ Уже сделано

  • NSI-формула исправлена. Метрика стресса теперь корректно растёт при росте пульса — больше не показывает 0 на демо-якоре. app/domain/hrv.py
  • IMU-пороги пересобраны (28/72 + гистерезис). Алгоритм перестал «прыгать» между rest / light / active — статус активности теперь стабильный, не дёргается на каждом микро-движении. imu_metrics.py
  • Слоты упрощены до staff + guest. Вместо трёх фиксированных слотов (Костя/Алексей/Гость) — один штатный + гость. Дашборд и mobile app говорят на одном языке. TeamSlotDashboard
  • Сегментация сессии при переподключении. Если браслет отвалился и переподключился — baseline для метрик берётся с нового сегмента, а не с начала сессии. Лечит «зачем у меня стресс 80, я же спокоен». pulse_session_service
  • Stale guest cleanup. Сессии без свежей телеметрии (браслет ушёл / гость снял) автоматически закрываются — слот гостя освобождается для следующего посетителя без ручной интервенции оператора.
  • Android foreground service. Пульс продолжает идти на бэк, когда экран mobile app выключен или приложение свёрнуто. Лечит главный факап стенда «пульс пропадает при выключенном экране».
  • BLE-фильтр VITRO-only + расширенный handshake (8 с). Скан больше не подтягивает чужие BLE-устройства (Valdus и пр.), а handshake выдерживает медленные ответы браслета — подключение надёжнее.
  • Bayevsky: poll R-R каждые 2 секунды. Mobile app не ждёт молча, пока браслет выгрузит интервалы — пингует статус и тянет данные сразу, как они готовы. Тест Баевского быстрее завершается. GET /v1/sessions/{id}/rr-status
  • 5 стресс-аватаров на kiosk. Zen / Samurai / Cheetah / Berserk / Volcano по результату Баевского — заменили технический «ИН 187» на образный «Гепард». assets/levels
  • Консолидация kiosk-фронта. Раньше было 2 параллельных версии (v1 и v2-personas) — выбрали v2 и убрали v1. Один источник правды, две прод-ссылки: /personas/ (плазма) и /kiosk/ (drill-down).
  • PDF-отчёт с интерпретацией аватаров и рекомендациями. Гость получает в TG не только число ИН и ярлык «Гепард», но и описание состояния + конкретные советы — закрывает критику Никиты «нет смысла оценки». Паша · TG-бот
  • Adaptive layout для kiosk-планшета. Карточка drill-down корректно отображается на экранах планшета, а не только на 1920×1080 — стенд перестаёт быть привязан к большой плазме. Кирилл · 03.06

● В работе

  • «Сон» в MVP. Цифры качества сна (Deep / REM / Light) сейчас выглядят некорректно — нужно разобраться, проблема на стороне SDK браслета или mobile app. Никита настаивает включить Сон в MVP — без него V2-фичи (Recovery, Burnout) не работают полноценно. Костя · 29.05
  • Дизайн mobile app для App Store / Google Play и B2C. Сейчас у нас инженерный UI, годный для команды и демо, но не для розничного покупателя. Нужен полный дизайн под публикацию в сторах и B2C-продажи (онбординг, экраны метрик, тариф, профиль). Костя · 29.05

◇ На поставщике / vendor

  • Vigor IMU Raw 50 Hz, 6 осей. Сейчас браслет отдаёт уже обработанный статус активности — для собственных алгоритмов (Activity, gestures, sleep-stages) нужен сырой поток акселерометра. Поставщик готов сделать за допразработку. письмо 03.06
  • Vigor PPG Raw 50 Hz. То же самое для оптического сенсора пульса — без сырого PPG мы не можем считать свой HRV-фильтр, SpO₂ и аритмии. Поставщик готов за допразработку. письмо 03.06
  • Vigor Bug 1: explicit BLE disconnect. Браслет не отключает предыдущий телефон при подключении к новому — отсюда главный факап стенда «три браслета одновременно не парятся». Поставщик подтвердил баг, ждём fix в прошивке.
  • Vigor Bug 3: erase pairing data on shutdown. Браслет сохраняет данные сопряжения после выключения, поэтому неочевидно, в каком он состоянии при включении — отсюда жалоба «нет никакой индикации, включён или нет». Ждём fix.
§3 · Уроки

Что не зашло

5 точек, где мы откровенно недодали. 3 продуктовых (интерпретация, спрос, hardware) и 2 процессных (release-checklist, status-page).

Продуктовые

5-аватарная метафора без интерпретации не считывается. Никита 28.05 22:40: «Нет описания смысла оценки. Что значит гепард, что с твоей нервной системой сейчас, какие рекомендации». Гость видит ярлык «Гепард», но не понимает, хорошо это или плохо и что с этим делать.

Лечение: PDF-отчёт с расширенной интерпретацией → в работе

Спрос B2B разошёлся с продуктом. Костя 28.05 13:32 после питча: «Хотят чтобы браслет бил током водителей. Плавание тоже хотят себе». Stress-management через мягкие нотификации vs запрос на punishment + waterproof — мы продаём не то, что просят.

Action: переоценить ICP для Беговой Лиги / транспортных компаний на M3 brainstorm

Молчание B2B post-Village. С 30.05 по 03.06 в рабочем чате — ни одного письма / лида / обратной связи от посетителей стенда. Косвенный сигнал, что воронка скромная или follow-up не выстроен.

Action: организовать ретро по лидам с Алексеем (B2B Lead)

Процессные

Нет release-checklist перед стендом. Костя 28.05 14:40 «Алексей обновлял, да», Кирилл 14:59 «сомнительно». На прод-устройстве у демо-якоря — старая APK, фича foreground-service в коде, но не в сборке.

Action: версионирование APK + ручная сверка commit-hash на устройствах за день до демо

Нет status-page и fallback-хостинга. Yandex Cloud положил статику personas на втором дне стенда. У команды нет ни мониторинга uptime, ни запасной площадки для статики (CDN / S3-зеркало).

Action: настроить простейший uptime-monitor + Cloudflare-зеркало для статики на M3
