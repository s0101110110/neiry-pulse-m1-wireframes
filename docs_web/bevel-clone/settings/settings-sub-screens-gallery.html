Возражения skill'ов — на разбор PM

Skills формально недоступны из этой cwd (docs_web/), методология применена вручную. Возражения, которые предположительно подняли бы redesign-existing-projects / impeccable typeset / impeccable harden:

impeccable typeset «Размер 15px для row-label — не в твоей шкале (cap 10 · sub 12 · body 13 · label 14 · subhead 18).» Решение оставил 15px намеренно — список настроек требует читаемости на 320px-экране, label 14px смотрелся бы тонко рядом с 56-пиксельной row-height. Trade-off зафиксирован — на разбор PM.
redesign-existing-projects «Floating pill-nav (5-tab круглые иконки) — отступление от base bevel-clone, где использовался полноширинный bottom-nav с подписями „Дом / Тренировка / История / Здоровье / Ещё“.» Брифом затребован 5-tab floating pill (Дом / История / Sharing / Настройки / Ещё). На разбор PM: сохраняем pill или возвращаемся к full-width nav?
impeccable harden «Pulse Mono подключён через JetBrains Mono web-fallback (Google Fonts), а не через локальные .woff2 как в DESIGN-bevel-clone.html.» Path ../UI_assets/fonts/pulse-mono/ из standalone-файлов docs_web/bevel-clone/settings/ требует ../../../UI_assets/fonts/... — путь сломается при перемещении файлов. JetBrains Mono (open-source SIL OFL, та же баз гарнитура) даёт идентичную глифику без зависимости от файловой структуры.
impeccable typeset «В Settings → BT иконки в row-icon показаны без круглой подложки (rounded square с tint) — это отступление от Bevel-style где settings-icon = 32×32 square 8px-radius.» Сознательный выбор: thin-line иконки без подложки дают более «тонкий» feel списка настроек, и согласуются с iOS Health/Apple Settings (raw line-icons + chevron). Bevel-style icon с tinted background оставлен только для device-card (Section 1, hero of list).
