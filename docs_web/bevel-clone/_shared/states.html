Логика капсулы

Одна формула для всех метрик. Капсула показывает позицию значения внутри личного коридора нормы.

position = (value − min) / (max − min) fill height = position · 100% · dot bottom = position · 100%
  • HRV 65.4 ms in [60, 100] → (65.4 − 60) / 40 = 14% → Ниже (warn)
  • RHR 58 bpm in [50, 70] → (58 − 50) / 20 = 40% → Норма (norm)
  • NSI 78 in [0, 100] → 78 / 100 = 78% → Выше (warn)
  • Сон 7ч 12м (7.2ч) in [5, 10] → (7.2 − 5) / 5 = 44% → Норма (norm)
  • Темп 38.1 °C in [35.5, 38.5] → (38.1 − 35.5) / 3 = 87% → Внимание (high)
Path B · инвариантная ориентация (принято). Капсула всегда заполняется снизу вверх по формуле value-from-bottom, независимо от того «выше лучше» или «ниже лучше» для метрики. Семантику «хорошо / плохо» несёт цвет статуса (green/orange/red), не ориентация заливки. Так одна формула работает для всех метрик (RHR 40% = green «ниже лучше», NSI 78% = orange «выше хуже»), разработчику предсказуемо и без инверсий (Path A отклонён).
